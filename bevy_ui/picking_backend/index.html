A picking backend for UI nodes.

§Usage

This backend does not require markers on cameras or entities to function. It will look for any pointers using the same render target as the UI camera, and run hit tests on the UI node tree.

§Important Note

This backend completely ignores FocusPolicy. The design of bevy_ui’s focus systems and the picking plugin are not compatible. Instead, use the optional PickingBehavior component to override how an entity responds to picking focus. Nodes without the PickingBehavior component will still trigger events and block items below it from being hovered.

§Implementation Notes

  • bevy_ui can only render to the primary window
  • bevy_ui can render on any camera with a flag, it is special, and is not tied to a particular camera.
  • To correctly sort picks, the order of bevy_ui is set to be the camera order plus 0.5.
