System that runs on Update, calculating the Transform of light segments from the corresponding LightRaySource. Note that this calculation happens every frame, so instead of rapidly spawning/despawning the entities, we spawn them and cache them in the LightSegmentCache, then modify their Visibility and Transforms.

If needed, optimization work can be done by recalculating only segments that are currently changing (segments already “stabilized” usually won’t move).

Similar logic is duplicated in preview_light_path, these two systems should be merged.
